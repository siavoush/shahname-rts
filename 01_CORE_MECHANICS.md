# Core Mechanics — MVP Specification

*The mechanical heart of the Shahnameh RTS, scoped for what we can actually build.*
*Last updated: 2026-04-22*

---

## 0. Purpose

This document specifies the mechanical systems for the MVP (Tier 1 vertical slice). It is the spec that Claude Code will build against in the next phase. Decisions in this document are *committed* unless explicitly marked **TBD**.

Companion documents:
- `00_SHAHNAMEH_RESEARCH.md` — research, lore, and overall scoping (read first if new to the project)

This document deliberately avoids over-specification. RTS design at this stage is most usefully expressed as **rules and intent**, not numbers — the numbers will emerge from prototyping. Where numbers appear here, they are starting points to be tuned, not contracts.

---

## 1. MVP Definition

The MVP is **Tier 1: vertical slice** as defined in `00_SHAHNAMEH_RESEARCH.md` §10. Concretely:

**In scope (MVP must have):**
- One playable faction: **Iran**
- One AI opponent faction: **Turan** (no Farr/Zur mechanics — uses simple AI economy)
- One battle map (~medium-sized, mixed terrain)
- Iran's **Farr** system fully implemented as a civilization-level meter
- One Iran hero unit: **Rostam**
- ~6 unit types for Iran (see §6)
- ~5 unit types for Turan AI (functional, not yet asymmetric)
- 2 tech tiers per faction (Village → Fortress; the third tier "Royal Court" is post-MVP)
- Core loop: gather resources → build base → produce army → defeat enemy
- Win condition: eliminate enemy main building (the Throne)
- The **Kaveh Event** (Farr-collapse revolt) functional even if simplified

**Explicitly out of scope for MVP (do NOT build):**
- Multiple maps (one is enough to validate the loop)
- Multiplayer (single-player skirmish only)
- Campaign missions (post-MVP — the campaign comes at Tier 2/3)
- Pahlavan duels (signature mechanic — defer to Tier 2)
- Turan's Zur system (defer to Tier 2 when Turan becomes playable)
- Divs and Shar (defer to post-launch)
- Persian-language UI (start English-only, add Persian at Tier 2 — but UI strings should be externalized from day one to make the future addition cheap)
- Sound design beyond placeholder (audio is a Tier 2 concern)
- Save/load (don't bother for prototype matches)
- Multiple Iran heroes — Rostam only
- Simurgh summon, Derafsh-e-Kaviani banner, war elephants — all post-MVP

**Why this scope:** every item above can be a "would be cool to add" trap. The MVP exists to prove the *loop* works (resource → build → fight → Farr matters) on one map with one matched opponent. Anything beyond that is a Tier 2 or Tier 3 concern.

## 2. The Core Match Loop

A standard match (skirmish) flows:

1. **Spawn**: each player starts with one Throne building, three workers, and the hero (Rostam for Iran).
2. **Economy**: workers gather resources from map nodes (see §3).
3. **Construction**: build production structures, Farr-generating structures, and military structures.
4. **Tech-up**: spend resources to advance tier (Village → Fortress).
5. **Military**: produce units; field an army; engage the opponent.
6. **Farr management** (Iran only at MVP): your actions accumulate or drain Farr; sustain it through buildings and just play.
7. **Victory**: destroy the enemy Throne. (Alternative loss condition: see §10.)

A typical MVP match is targeted at **15–25 minutes**. This is shorter than AoE2 (30–60 min) and closer to SC2 (10–25 min) — appropriate for a tight, focused, faction-locked design.

## 3. Resources

MVP uses a small resource set — three is the sweet spot for SC2/C&C-grade depth without AoE-grade complexity.

| Resource | Source | Used for |
|---|---|---|
| **Coin (سکه, *sekkeh*)** | Mines on the map (limited nodes) | Most production (workers, buildings, common units) |
| **Grain (غلات, *ghallat*)** | Farms (player-built on fertile tiles) | Unit upkeep, tech upgrades |
| **Farr (فر, *farr*)** | See §4 — generated by buildings + actions, NOT mined | Tier-3 unlocks (post-MVP), gating for hero abilities, prevention of Kaveh Event |

Note: Farr is *not* a spendable resource in the SC2-mineral sense. It is a *condition meter* that gates abilities and triggers events. You do not spend Farr to produce units. You sustain Farr to keep your civilization functional.

**Why three resources, not two:** two resources (gold + something) is the SC2 model and works, but for an RTS with a *farming culture* on one side and a *steppe-raider culture* on the other (long-term Tier 2 vision), grain-as-second-resource sets up natural faction asymmetry later (Turan raids grain rather than producing it). Build the affordance now even though both factions farm at MVP.

## 4. Farr — Iran's Civilization Meter (Full Spec)

Farr is the central mechanical innovation of this RTS. It expresses the Shahnameh's own theory of just rule as a playable system.

### 4.1 What Farr is

Farr is a **single empire-wide meter** for the Iran faction. Range: **0 to 100**. Starting value: **50** (neutral).

It is NOT attached to any specific unit, building, or hero. It is a property of your civilization as a whole, displayed prominently in the HUD as a circular gauge with a Persian-tile-pattern frame.

### 4.2 What Farr does

Farr governs three things:

1. **Tier gating**: Tier 2 (Fortress) requires Farr ≥ 40 to advance to. Tier 3 (Royal Court, post-MVP) will require Farr ≥ 70. Tiering down (because Farr drops) is *not* automatic — once advanced, you keep your tier. But you lose access to Tier 2 production until Farr recovers.
2. **Hero abilities**: Rostam's high-tier abilities (e.g., the rumored Simurgh summon, post-MVP) are gated by Farr thresholds.
3. **The Kaveh Event** (§9): if Farr drops below 15 and stays there, a popular revolt triggers.

### 4.3 What changes Farr

**Generators (Farr accumulates):**
- **Building Farr structures** — each generates passive Farr/min (see §5):
  - Atashkadeh (fire temple): +1 Farr/min
  - Dadgah (hall of justice): +0.5 Farr/min
  - Barghah (royal audience hall): +0.5 Farr/min
  - Yadgar (hero monument): +0.25 Farr/min, only after a hero has died once
- **Just actions** (one-time events):
  - Hero rescues another unit from death (last-second save): +3 Farr
  - Hero spares a defeated enemy hero: +5 Farr (post-MVP, requires Piran)
  - Player wins a match without using mass-civilian-killing tactics: +10 Farr (end-of-match bonus)

**Drains (Farr is lost):**
- **Unjust actions**:
  - Worker killed while idle and unarmed: −1 Farr each
  - Killing a Turanian civilian/worker (post-MVP, when civilians exist): −2 Farr each
  - Hero attacks an ally unit (yes, friendly fire counts): −5 Farr
  - Hero killed while fleeing combat: −10 Farr (Rostam's death must always be facing the enemy)
- **Cosmic events**:
  - Solar eclipse / sandstorm event in late-game: temporary Farr drain while it lasts
  - Loss of an Atashkadeh building: −5 Farr (the sacred flame is extinguished)

**Snowball protection (the Shahnameh anti-tyranny clause):**
- Killing a unit when your army outnumbers theirs by 3:1 or more: −0.5 Farr per kill (overwhelming victory does not earn glory)
- Destroying enemy economy (workers, mines) when their military is broken: −1 Farr per worker

This is the mechanic that prevents Farr from positive-feedback-looping. *You cannot bully your way to high Farr.* Crushing a defeated opponent drains it. This forces mid-match decisions that have moral and mechanical weight: *do I take the easy snowball, or play the longer, more honorable game that actually wins my civilization Farr?*

### 4.4 Visualization

The Farr gauge is the **most important UI element on the screen** after the resource counters. Requirements:

- Always visible in HUD top-right
- A visible *change feedback* every time Farr moves: small +/− numbers floating from the gauge with a brief reason text ("Hero rescued Savar: +3 Farr", "Worker killed defenseless: −1 Farr"). Every change is traceable.
- Color shifts: ≥70 gold/glow, 40–70 ivory, 15–40 dim, <15 red and pulsing
- A subtle audio cue when crossing threshold: chime up when crossing 40 or 70, distant horn down when crossing 15

The user-facing concept: *you should always know your Farr, why it changed, and what's at stake if it drops further.*

## 5. Buildings (Iran, MVP)

Two tiers. Construction by workers, takes time + resources.

### Tier 1 (Village) — available from start

| Building | Purpose | Cost (initial guess, tune in playtesting) |
|---|---|---|
| **Throne (تخت)** | Capital. Loss = defeat. Spawns workers. Comes pre-placed at game start. | — |
| **Khaneh** (house) | Population cap +5 per building | 50 coin |
| **Mazra'eh** (farm) | Generates grain | 60 coin |
| **Ma'dan** (mine) | Worker-staffed coin extraction at mine nodes | 40 coin |
| **Sarbaz-khaneh** (barracks) | Produces Tier 1 military: Piyade, Kamandar | 100 coin |
| **Atashkadeh** (fire temple) | +1 Farr/min, prerequisite for Tier 2 advance | 150 coin, 50 grain |
| **Dadgah** (justice hall) | +0.5 Farr/min, also enables hero respawn | 200 coin, 75 grain |

### Tier 2 (Fortress) — requires Farr ≥ 40 + Atashkadeh built

| Building | Purpose | Cost |
|---|---|---|
| **Qal'eh** (fortress, the tier-up itself) | The Fortress structure converts your Throne to "Fortress mode," unlocking Tier 2 buildings | 400 coin, 200 grain, requires Atashkadeh |
| **Sowari-khaneh** (cavalry stable) | Produces Savar (cavalry) | 200 coin |
| **Tirandazi** (archery range) | Produces advanced Kamandar variants | 175 coin |
| **Barghah** (royal audience hall) | +0.5 Farr/min, enables Rostam to be fielded if not already; hero respawn site | 300 coin, 100 grain |
| **Yadgar** (hero monument) | +0.25 Farr/min, only after a hero dies. Built at site of hero's last death | 150 coin |

Tier 3 (Royal Court) is **post-MVP**.

## 6. Units (Iran, MVP)

Six unit types. Deliberately small roster — every unit must have a clear role and a clear silhouette.

### Workers (production)
- **Kargar** (worker) — gathers resources, constructs buildings, repairs. Combat: minimal. Fragile.

### Tier 1 military
- **Piyade** (foot infantry) — cheap, slow, durable. Spear-and-shield. Strong vs. cavalry, weak vs. archers.
- **Kamandar** (archer) — ranged infantry. Standard Persian/Parthian archer. Strong vs. infantry, weak in melee.

### Tier 2 military
- **Savar** (cavalry / knight) — fast melee. Strong vs. archers, weak vs. spear-piyade. The classic Persian cavalry.
- **Asb-savar Kamandar** (horse archer) — mounted ranged. Mobile, kiting unit. Expensive. The Parthian-shot unit.

### Hero
- **Rostam** — see §7.

This is a **rock-paper-scissors** triangle: piyade > savar > kamandar > piyade. Horse archers complicate it (kiting). Rostam transcends it (hero unit).

## 7. Rostam — Hero Unit (Iran, MVP)

The single hero on the Iran side. Treat him as the spec for "what a Shahnameh hero unit is."

### Stats (starting values, to tune)

- **Health**: ~10× a regular Savar
- **Damage**: ~5× a regular Savar, area-of-effect on melee swing (sweeping cleave)
- **Speed**: equal to mounted units (Rakhsh)
- **Population cost**: 0 (heroes don't count against pop cap)
- **Build cost**: not built normally — see §7.4

### 7.1 Visual identity

- Mounted on **Rakhsh** (his legendary horse) — visually inseparable. Dismount is rare and dramatic.
- Distinctive silhouette: heavier than a regular savar, golden highlights, leopard-skin (palangineh) cloak.
- Voice: low, calm, weight. (Voice acting is post-MVP, but stub the line system in.)

### 7.2 Active abilities (MVP — keep small)

Two abilities, manually triggered:

- **Cleaving Strike** (cooldown ~30s): swings Rakhsh into a charging attack that damages all enemies in a wide arc in front. Ability of the legendary hero, not a regular cavalry charge.
- **Roar of Rakhsh** (cooldown ~60s): nearby allied units gain a temporary attack speed and morale buff (+25% attack speed for 8s). Hero presence emboldens his army — Shahnameh-canonical.

Post-MVP additions: Bow shot (range), Simurgh summon (if Farr is high), Champion Duel (challenge enemy hero).

### 7.3 Death and respawn

- Rostam can die. He dies dramatically — slow-motion fall, screen shake, distant lament.
- On death, an immediate **Farr penalty**: −5 if killed in honest battle, −10 if killed while fleeing.
- **Respawn** at the Dadgah (Tier 1) or Barghah (Tier 2) after a long cooldown — start with **120 seconds**. Tunable. Respawn cost: significant coin + grain.
- After his first death, a **Yadgar** building can be constructed at the death site (extra Farr generation). This is canon: the Shahnameh records monuments to fallen heroes throughout.

### 7.4 Spawning / availability

At MVP: Rostam spawns at game start, free of charge, alongside the player's starting workers and Throne. No "build Rostam" decision. He is your hero from minute one.

(Post-MVP, when multiple heroes exist: the player will choose at match start which hero to field, and additional heroes become tier-gated unlocks.)

## 8. Tech Tiers — How Iran Advances

**Two tiers at MVP** (the third comes later):

### Tier 1 — Village
- Default starting tier
- All Tier 1 buildings available
- Piyade and Kamandar producible

### Tier 2 — Fortress
- **Requirements**: Atashkadeh built, Farr ≥ 40, 400 coin + 200 grain
- Construction time: 90 seconds
- Unlocks: Sowari-khaneh, Tirandazi, Barghah, Yadgar (Yadgar requires hero death first)
- Unlocks: Savar and Asb-savar Kamandar production
- Visual: your Throne building gains Tier-2 architectural details (more pillars, larger roof, gold leaf accents)

### Tier 3 — Royal Court (post-MVP)
- Requirements (planned): Barghah built, Farr ≥ 70, very expensive
- Will unlock: Simurgh summon, Derafsh-e-Kaviani banner, war elephants, Tier-3 hero abilities

## 9. The Kaveh Event — Farr Collapse Mechanic

The single most distinctive mechanic in the game. **The reason your civilization can fall apart from within.**

### 9.1 Trigger

When **Iran's Farr drops below 15** AND remains below 15 for **30 continuous seconds**, the Kaveh Event triggers.

The 30-second grace period exists so the player has a chance to recover — emergency construction of an Atashkadeh, a noble action that restores Farr — before the event fires.

### 9.2 What happens

When the event triggers:

1. **Visual/audio**: dramatic music sting, screen tint shift toward red/gold, a banner unfurls in the world (the Derafsh-e-Kaviani — Kaveh's apron-banner — raised against your tyranny)
2. **A Kaveh hero unit spawns at your Throne**, hostile to you, leading **a small rebel army** (4–6 piyade-equivalent units)
3. **All your workers stop working for 15 seconds** (general strike) and may defect (~25% per worker chance) to join the rebel force
4. **Your Farr is locked at its current value for 60 seconds** — you can't easily restore it during the event

### 9.3 Resolving the event

You can survive the Kaveh Event in two ways:

- **Defeat Kaveh and the rebel force in combat** (the harsh path) — restores normal play but Farr remains low; another event will trigger if Farr stays under 15
- **Restore Farr above 30 within 90 seconds** (the just path) — Kaveh and rebels disband peacefully, your civilization is reconciled. This is mechanically harder (Farr restoration takes time and just actions) but narratively the *correct* Shahnameh response

The latter is better for the player long-term but costs short-term economic productivity.

### 9.4 Why this matters

This mechanic is the proof that Farr is not a flavor system but a *real strategic force*. Players who ignore Farr risk an in-game uprising that can lose them the match. It also creates one of the most narratively resonant moments in any RTS — Kaveh's banner rising against your unjust rule, drawn directly from the founding act of revolt in the epic.

It is also the mechanic most likely to need tuning. Start with the values above; expect to adjust thresholds and durations significantly through playtesting.

## 10. Win and Loss Conditions

### Victory (one of):
- **Destroy the enemy Throne building** (primary win condition)
- **Eliminate all enemy units and buildings** (alternate, almost always equivalent to the first)

### Defeat (one of):
- **Your Throne is destroyed**
- **You lose all units and buildings** (effectively impossible without first losing the Throne)
- **(Optional, post-MVP)**: surrender — the Kaveh Event escalates if Farr stays catastrophically low; eventually civilization collapses

There is **no time limit**. Matches end when one side wins.

## 11. UI / Readability Requirements

A focused list of UI elements that absolutely must work for the MVP. Each is critical to playability.

- **Resource counters** (top-left): Coin, Grain, current Farr value, current population / cap
- **Farr gauge** (top-right): circular meter, 0–100, with color thresholds and threshold-crossing audio cues
- **Farr change feed**: small floating numbers from the gauge, each labeled with the cause
- **Tier indicator**: visual in the HUD showing current tier (Village / Fortress)
- **Selected unit panel** (bottom-left): unit portrait, stats, abilities
- **Build menu** (bottom-right): contextual to selected unit/building
- **Minimap** (bottom-center or bottom-right): standard RTS minimap with fog of war
- **Hero portrait** (always visible): Rostam's portrait shows his current health, ability cooldowns, status — clickable to center camera on him
- **Kaveh Event warning**: when Farr drops below 25, a subtle warning indicator. Below 20, urgent warning. Below 15, the 30-second countdown begins visually

**The rule**: every game state the player needs to make decisions about must be visible at a glance. If a player has to ask "wait, what's my Farr?", the UI has failed.

## 12. AI Opponent (Turan, MVP)

For MVP, Turan is **AI-only** with a **simplified economy** (no Zur). The AI doesn't have to be brilliant — it has to be *playable*, meaning it produces a reasonable opposing army and engages the player.

### Turan AI requirements (MVP)

- Builds workers, gathers resources at standard rates
- Constructs buildings in a fixed sensible order
- Tech-ups to Fortress at a fixed time threshold (~5 minutes into match)
- Produces a mixed army of Turan unit types (parallel to Iran's units — same archetypes, slightly different stats for asymmetry-of-feel)
- Has Piran as a hero unit that joins the army (full Piran spec deferred to Tier 2)
- Attacks the player periodically — small early skirmishes, larger mid-game pushes, attempts to destroy the Throne in late-game
- **Three difficulty levels**: Easy, Normal, Hard. Differ primarily in resource bonuses and aggression timing, not in AI sophistication

### Explicit AI non-goals (MVP)

- Not strategic (does not adapt to player choices)
- No diplomacy or surrender
- No use of any Zur mechanic — Turan's mechanic is reskinned-standard for now

This is enough to playtest the Iran loop. Turan's full spec comes when we make Turan playable in Tier 2.

## 13. Open Design Questions

Things deliberately left unresolved at this stage. Each should be answered through prototyping, not specification:

1. **Map size and density** — TBD until we prototype movement and combat. Goal: matches average 15–25 minutes.
2. **Camera**: 2.5D isometric (fixed angle, no rotation) vs. low-angle 3D with rotation. Decide at Tier 0 prototype based on what feels best.
3. **Unit counts at full population** — TBD. Initial guess: pop cap ~120 per side, so ~80 military units in a max army. Tune based on what reads well at standard zoom.
4. **Worker auto-tasking** — modern QoL feature (workers automatically gather nearest resource). Probably yes, but decide after first prototype.
5. **Should Farr decay passively over time?** — probably no for MVP (too punishing for new players), but worth experimenting with later.
6. **Hero respawn cost** — 120s is a starting guess; might be too short or too long.

## 14. Implementation Notes for Claude Code (when we get there)

When this doc becomes code, observe these:

- **Externalize all gameplay constants** in a single `constants.gd` file. The numbers in this doc *will* change in playtesting.
- **All UI strings** in a translation table from day one, even if only English is filled in. Persian addition (Tier 2) becomes trivial.
- **Farr changes** flow through a single function (`apply_farr_change(amount, reason_string, source_unit)`) — every Farr movement goes through here so logging, UI feedback, and event triggers are centralized.
- **Save the design rationale alongside the code** — when adding a new ability to Rostam or a new Farr-changing event, add a comment with the Shahnameh source for it. The setting is the project's identity; keep it in the code's bones.
